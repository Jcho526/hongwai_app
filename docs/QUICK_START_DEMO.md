
```javascript

```

```bash

# 重新编译

# 菜单 → 运行 → 运行到真机设备
```

## 🔍 功能详细说明

### 功能1：自动启动远程App

```javascript
// 当进入实时监控页面时自动执行
async initRemoteScreencast() {
  // 1. 检查远程App是否已运行
  const isRunning = await appLauncher.isTargetAppRunning();
  
  // 2. 如果未运行，则自动启动
  if (!isRunning) {
    console.log('启动远程应用...');
    await appLauncher.launchTargetApp();  // ← 自动启动
  }
  
  // 3. 启动屏幕投射
  await appLauncher.startScreencast();
}
```

**用户体验**：
```
用户点击"实时监控"
    ↓ 0秒
页面加载完成，显示"正在连接..."
    ↓ 1秒

    ↓ 3秒

    ↓ 5秒
远程App成功登录，屏幕投射到右侧
    ↓ 7秒
用户可以看到双屏界面
```

### 功能2：屏幕投射（3种方案）

#### 方案A: WebRTC（最优秀 🌟）
```
优点：
- 实时性好（延迟<100ms）
- 视频质量高
- 支持音频传输

缺点：
- 需要信令服务器
- 实现复杂

性能：
- 帧率: 30fps
- 延迟: <100ms
```

#### 方案B: ADB截屏轮询（推荐 ⭐）
```
优点：
- 实现简单
- 不需要服务器
- 兼容性好

缺点：
- 帧率较低（2fps）
- 延迟较大（500ms）

性能：
- 帧率: 2fps (每500ms刷新一次)
- 延迟: 200-500ms
```

#### 方案C: 摄像头直播（简化版）
```
优点：
- 实现最简单
- 帧率可以高

缺点：
- 只支持摄像头画面
- 无法看到UI按钮

性能：
- 帧率: 30fps
- 延迟: <200ms
```

**自动降级流程**：
```
尝试WebRTC
    ↓ (如果失败)
尝试ADB截屏
    ↓ (如果失败)
使用摄像头直播
    ↓ (都失败)
提示用户连接失败
```

### 功能3：触摸转发

**工作原理**：
```
用户点击右侧屏幕
    ↓
获取点击坐标 (500, 400)
    ↓
换算到远程分辨率 (1920, 1440)
    ↓
通过ADB发送点击命令
    ↓
远程App响应点击
```

**坐标转换公式**：
```javascript
// 容器大小: 960 x 540
// 远程分辨率: 1920 x 1080

scaledX = Math.floor(relativeX * (1920 / 960));  // = relativeX * 2
scaledY = Math.floor(relativeY * (1080 / 540));  // = relativeY * 2
```

**触摸事件类型**：
```javascript
touchDown(x, y)   // 手指按下
touchMove(x, y)   // 手指移动
touchUp(x, y)     // 手指抬起
```
